Hypertext Transfer Protocol(HTTP)

Definition

the "language" or set of rules that browsers and web servers use to talk to each other. HTTP is the delivery truck moving the data back and forth enabling data exchange on web.

HTTP works like a conversation between two parties; The Request (Browser): the browser sends a message to the server and The Response (Server): The server looks for the file and sends a message back.

HTTP Status Codes- every time a server responds, it includes a 3-digit code to tell the browser how it went eg 200 OK, 404 Not Found, 301/302 Redirect.

HTTP has methods that include:

  • GET: Ask for data (like viewing your index.html).
  • POST: Send data to a server (like submitting a "Contact Me" form).
  • PUT/PATCH: Update existing data.
  • DELETE: Remove data.

HTTP vs. HTTPS: In standard HTTP, data is sent in "plain text." If you were on a public Wi-Fi, a hacker could "see" what you are typing while HTTPS uses SSL/TLS encryption to scramble the data so only the server and your browser can read it. The "S" stands for Secure.
