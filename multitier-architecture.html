multitier-architecture

Definition

multitier-architecture is a system that is deployed on multiple different nodes eg presentation, logic and data.

multitier-architecture is a way of organizing a software system by breaking it into separate "layers" or "tiers." Each tier has a specific job and only talks to the tiers directly above or below it.

Reasons for using tiers is for scalability, security and maintenance. Below is the 3-Tier Architecture:

  • Presentation Tier- This is the part the user actually sees and interacts with eg web browser which displays information to the user and collect their input (like clicking a button or typing in a form).
  • Application/Logic Tier- The "brain" of the operation. This is where the code lives that makes decisions eg java server which It processes the data from the user. For example, if you log in, this tier checks if your password is correct. It acts as the middleman between the user and the data.
  • Data Tier (The Database)- The storage room eg MySQL which stores and manages the actual data (user profiles, product lists, etc.) so it can be retrieved later.
